# Dev Log — 2026-04-15

## Summary

Full session shipping the **Unified Inbox** feature (Phase 2). This is a fully AI-powered, multi-channel inbox that aggregates conversations from Gmail, LinkedIn (HeyReach), ManyChat (Instagram/Facebook), and Website Forms into a single 3-panel UI. AI classifies every conversation as Prospect / Not Prospect / Warmup using Claude or GPT. Users can reply manually or via AI-assisted draft. Warmup emails from Instantly, Smartlead, Lemwarm etc. are auto-filtered. Build passes with zero TypeScript errors.

### DB Migration — `supabase/migrations/011_inbox.sql`

Four new tables:

**`inbox_accounts`** — Connected channels per user
- `provider`: `gmail` | `linkedin` | `manychat` | `form`
- `account_label`: friendly name (supports multiple Gmail accounts)
- `email`: for Gmail accounts
- `access_token_encrypted`, `refresh_token_encrypted`: AES-256 (same encryption as API keys)
- `extra_config JSONB`: provider-specific extras (webhook_token for forms, masked_key for API-key providers)
- `last_synced_at`: for sync tracking

**`inbox_conversations`** — Thread-level groupings
- `classification`: `prospect` | `not_prospect` | `warmup` | `unclassified`
- `classification_reason`, `classification_score` (0–100): AI output stored per conversation
- `is_read`, `is_archived`, `is_blocked`: state flags
- `contact_name`, `contact_email`, `contact_company`, `contact_linkedin_url`
- Unique constraint on `(account_id, external_thread_id)` for dedup

**`inbox_messages`** — Individual messages
- `direction`: `inbound` | `outbound`
- `body`, `body_html`: message content
- `sent_at`: actual message timestamp

**`inbox_settings`** — Per-user AI config (UNIQUE per user)
- `icp_description`: free-text ICP for AI context
- `sales_keywords[]`: default `['proposal', 'demo', 'appointment', 'meeting', 'pricing', 'quote', 'interested', 'schedule', 'call']`
- `blocked_senders[]`: email addresses or domains to block
- `block_warmup_tools`: auto-filter warmup signals (default true)
- `auto_classify`: classify on arrival (default true)
- `ai_provider`: `anthropic` | `openai`

All tables have RLS policies (`user_id = auth.uid()`).

> **Action required:** Run `supabase db push` to apply migration `011_inbox.sql`.

### API Routes

#### Accounts

**`GET /api/inbox/accounts`**
- Returns all active connected accounts for the user

**`POST /api/inbox/accounts`**
- Body: `{ provider, account_label, email?, api_key? }`
- For API-key providers (LinkedIn/ManyChat): encrypts key with `encryptApiKey()`
- For `form` provider: generates a `crypto.randomUUID()` webhook token stored in `extra_config`
- For OAuth providers (Gmail): redirected through OAuth flow instead

**`PATCH /api/inbox/accounts/[id]`**
- Toggle `is_active` or rename `account_label`

**`DELETE /api/inbox/accounts/[id]`**
- Disconnects account (cascades to conversations + messages via FK)

#### Conversations

**`GET /api/inbox/conversations`**
- Query params: `filter` (all/prospect/not_prospect/warmup/unread/archived), `account_id`, `page`, `limit`
- Joins `inbox_accounts` for provider/label display
- Always excludes `is_blocked = true`

**`POST /api/inbox/conversations`**
- Manually create a conversation + first inbound message (used by form webhooks)

**`GET /api/inbox/conversations/[id]`**
- Returns full conversation with all messages ordered by `sent_at`
- Auto-marks `is_read = true`

**`PATCH /api/inbox/conversations/[id]`**
- Update `is_read`, `is_archived`, `is_blocked`

**`POST /api/inbox/conversations/[id]/classify`**
- Fast-paths (no AI call):
  1. Blocked sender check (email/domain against `blocked_senders[]`)
  2. Warmup signal detection (checks for Instantly, Smartlead, Lemwarm, Mailreach, Warmbox phrases in subject + body)
- AI path: sends thread to Claude `claude-haiku-4-5-20251001` or `gpt-4o-mini` with system prompt including ICP + sales keywords
- Returns JSON `{ classification, reason, score }` and updates the conversation row

**`POST /api/inbox/conversations/[id]/reply`**
- `mode: "draft_ai"` → generates a reply draft using Claude Sonnet or GPT-4o (reads full thread context)
- `mode: "send"` → saves outbound message to `inbox_messages`; actual delivery (Gmail API, ManyChat) to be wired per provider

#### Settings

**`GET /api/inbox/settings`** — returns settings row or defaults

**`POST /api/inbox/settings`** — upsert on `user_id` conflict

### Gmail OAuth — `GET /api/oauth/gmail` + `GET /api/oauth/gmail/callback`

- Same PKCE-style + HMAC state pattern as Reddit / Twitter OAuth
- State encodes `user_id:timestamp:label` (base64) so the callback knows which account label the user entered
- Scopes: `gmail.readonly gmail.send userinfo.email`
- `access_type: offline, prompt: consent` to get refresh token
- Callback exchanges code → tokens → fetches Google profile email → stores encrypted in `inbox_accounts`
- CSRF: `gmail_state` httpOnly cookie (10 min TTL), cleared on success
- Redirects to `/inbox?connected=gmail` on success

**New env vars needed:**
```
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
```

## Classification Logic

### Warmup Auto-detection (no AI cost)
Checks subject + body for signals from: Instantly, Smartlead, Lemwarm, Mailreach, Warmbox, plus generic phrases like "you're one of the great" and "this is a warmup email".

### AI Classification Prompt
- System: includes ICP description + sales keywords + classification rules
- Model: `claude-haiku-4-5-20251001` (Anthropic) or `gpt-4o-mini` (OpenAI) — cheap/fast for classification
- Returns: `{ classification, reason, score }` JSON
- Reply drafts use `claude-sonnet-4-6` / `gpt-4o` — higher quality for composing
