# Dev Log — 2026-04-24

## Summary

Shipped **OutboundSync integration** (CRM sync). OutboundSync (https://outboundsync.com) is a one-way relay that receives webhooks from SEPs (Instantly, Smartlead, HeyReach, EmailBison) and writes them into a CRM (HubSpot, Salesforce, Close). Integration is two-way for CampaignFlow Pro users:

1. **Auto-wire** — on campaign launch into Instantly / Smartlead / Heyreach, we programmatically register the user's OutboundSync per-SEP webhook URL on the SEP campaign (idempotent: skips if URL already present). Best-effort — never blocks campaign launch.
2. **Forward** — when the unified inbox classifies a reply as `prospect`, we POST a CampaignFlow-native event (HMAC-SHA256 signed) to the user's OutboundSync generic webhook URL so the reply lands in their CRM even if the SEP's own webhook missed it.

No new tables, no new migrations, no new cron jobs. Reuses the existing `integration_configs` table + `getGlobalApiConfig` helper. Typecheck passes clean.

## Feature — OutboundSync Integration

### Architecture

```
CampaignFlow → push-leads → Instantly/Smartlead API
                                 ↓
                          ensureOutboundSyncWebhook(sep, campaignId, apiKey, url)
                                 ↓
                   POST SEP webhook config → OutboundSync URL registered

CampaignFlow → social launch → Heyreach list
                                 ↓
                          ensureOutboundSyncWebhook("heyreach", listId, apiKey, url)

Unified Inbox → classify → "prospect"
                                 ↓
                   forwardEventToOutboundSync(supabase, userId, event)
                                 ↓
                   HMAC-signed POST → OutboundSync generic URL → user's CRM
```

### Credential storage

Single row in `integration_configs` with `service = "outboundsync"`, `api_key_encrypted` = AES-256 ciphertext of a JSON blob:

```json
{
  "signing_secret": "...",
  "instantly_webhook_url": "https://hooks.outboundsync.com/instantly/...",
  "smartlead_webhook_url": "https://hooks.outboundsync.com/smartlead/...",
  "heyreach_webhook_url":  "https://hooks.outboundsync.com/heyreach/...",
  "generic_webhook_url":   "https://hooks.outboundsync.com/generic/..."
}
```

Retrieved via the existing [`getGlobalApiConfig`](src/lib/api/get-integration-config.ts:11) helper (auto-parses JSON multi-field blobs).

### Settings UI

New tile in `INTEGRATIONS` map in [GlobalIntegrationsForm.tsx](src/components/settings/GlobalIntegrationsForm.tsx), `usedIn: "CRM Sync"`, `multiField: true`. Five fields — HMAC signing secret (required, secret) + four optional webhook URLs (one per SEP + one generic). User pastes only what they need; blanks are skipped on save.

### SEP webhook auto-registration

Per-SEP sub-handlers in [`src/lib/integrations/outboundsync.ts`](src/lib/integrations/outboundsync.ts):

- **Instantly:** `GET/POST /api/v2/campaigns/{id}/webhooks` (Bearer auth). Events: `reply_received`, `email_bounced`, `unsubscribe`.
- **Smartlead:** `GET/POST /api/v1/campaigns/{id}/webhooks?api_key=X`. Events: `EMAIL_REPLY`, `EMAIL_BOUNCE`, `LEAD_UNSUBSCRIBED`.
- **HeyReach:** `GET /api/public/v1/webhooks/GetAll` → `POST /Create` with `listId` scope (X-API-KEY header). Events: `MESSAGE_REPLY`, `CAMPAIGN_LEAD_STATUS_CHANGED`.

All three list existing webhooks first and skip if the OutboundSync URL is already registered (idempotent). `ensureOutboundSyncWebhook` returns `{ registered, skipped?, error? }` — callers never throw.

### Event forwarding (prospect classifier → OutboundSync)

`forwardEventToOutboundSync()` in `outboundsync.ts` builds a canonical payload:

```json
{
  "source": "campaignflow",
  "occurred_at": "2026-04-24T...",
  "event_type": "reply.classified_interested",
  "contact": { "email", "first_name", "last_name", "company" },
  "context": { "conversation_id", "subject", "classification", "classification_score", "reply_preview" }
}
```

Computes HMAC-SHA256 over the serialized body using `signing_secret`, POSTs with:
- `X-CampaignFlow-Source: campaignflow-inbox`
- `X-CampaignFlow-Signature: sha256=<hex>`

Called fire-and-forget from [classify/route.ts](src/app/api/inbox/conversations/[id]/classify/route.ts) only when `classification === "prospect"` — adds zero latency to the classifier response.

## Trade-offs / notes

- **No OutboundSync public API** → we push one way only (SEP → OutboundSync via SEP webhooks, or CampaignFlow → OutboundSync via our generic webhook). We can't read from OutboundSync.
- **Generic payload shape** is ours, not OutboundSync's. Users set up an OutboundSync rule that accepts the `X-CampaignFlow-Source: campaignflow-inbox` header. Refine shape once tested against a live OutboundSync workspace.
- **Best-effort auto-wire.** SEP webhook-registration failures (rate limits, permission issues) are surfaced under `results.outboundsync_webhook_{sep}` in the push-leads response but never block campaign launch.
- **Smartlead campaign_id source** — pulled from the Smartlead integration config itself (not the push result), since Smartlead's `campaign_id` is stored per-user alongside the API key.
