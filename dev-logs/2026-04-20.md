## Production Deployment — Root Cause & Fix

The live site (`campaignflowpro.com`) had been stuck on commit `6530fe4` for days. Every deploy since then was ERROR. Root causes identified and fixed in sequence:

### 1. HubSpot template literal circular reference (TypeScript)
- **File:** `src/app/api/lists/scrape/route.ts` line 290
- **Bug:** `const afterParam = membAfter ? \`&after=${membAfter}\` : ""` inside a `do-while` loop triggered TypeScript circular inference (`'afterParam' implicitly has type 'any'`)
- **Fix:** Added explicit `: string` type annotation → `const afterParam: string = ...`
- **Commit:** `01ab911`

### 2. Anthropic SDK too old for `thinking: { type: "adaptive" }`
- **Bug:** `@anthropic-ai/sdk` v0.36.3 did not expose `thinking` types on `client.messages.create()`. The Super Agent's `agent-loop.ts` used `thinking: { type: "adaptive" }` which caused TypeScript error `'thinking' does not exist in type 'MessageCreateParamsNonStreaming'`
- **Fix:** Upgraded SDK to v0.90.0, switched to `client.beta.messages.create()` with beta header `interleaved-thinking-2025-05-14`
- **Commit:** `c9055a9`
- **Result:** First successful production deploy in days → all features (Automations, Billing history, social campaign stats, TopBar credit alerts, Super Agent) went live

## Full Feature Audit

Ran 3 parallel code audit agents across all features (Dashboard, Campaigns, Lists, Reports, Super DM Setter, Billing, Settings, Inbox, Automations, Super Agent). Four real bugs confirmed and fixed:

### Bug 1 — Toast event bus split (TopBar + AutomationsClient)
- **Files:** `src/components/dashboard/TopBar.tsx`, `src/components/automations/AutomationsClient.tsx`
- **Bug:** Both imported `toast` from `@/hooks/useToast` — a dead event bus. The root layout's `<Toaster>` only subscribes to `@/components/ui/toast`. Result: credit alerts and automation toasts were silently dropped
- **Fix:** Changed both imports to `@/components/ui/toast`

### Bug 2 — Campaign report page: allCampaigns query missing join
- **File:** `src/app/(dashboard)/campaigns/[campaignId]/reports/[reportId]/page.tsx` lines 83-88
- **Bug:** Query filtered on `clients.user_id` without including `clients` in the select — PostgREST can't filter on an unjoined table, so `allCampaigns` returned null and global campaign totals showed as zero
- **Fix:** Added `clients!inner(user_id)` to the select clause

### Bug 3 — Inbox reply: invalid RPC inside update()
- **File:** `src/app/api/inbox/conversations/[id]/reply/route.ts` line 121
- **Bug:** `message_count: supabase.rpc("increment", { row_id: id }) as unknown as number` — RPC returns a Promise, not a number; storing it in an update column corrupts the value
- **Fix:** Removed RPC from update, split into separate `update()` for timestamps

### Bug 4 — Inbox reply: missing sender metadata
- **Same file**, lines 104-111
- **Bug:** Outbound message insert omitted `sender_name` / `sender_email`; select returned only 4 fields, missing fields the UI expected (causing blank sender display in thread view)
- **Fix:** Added `sender_name` / `sender_email` to insert using `user.user_metadata?.full_name`; expanded select to return all required fields

- **Commit:** `ea82f6f`
