### HubSpot Pagination — Data Loss on Lists >100 Contacts
**File:** `src/app/api/lists/scrape/route.ts` (lines 286–308)

**Before:** Single `limit=100` membership fetch + single batch/read call with `.slice(0, 100)` → silent data loss on any list with >100 contacts.

**After:**
- Memberships endpoint paginated via `after` cursor (up to 500 total)
- Batch/read loops in 100-contact chunks
- All contacts collected before returning

## Super AI Agent — Design & Planning

Designed the next major feature. Full spec documented below.

### What It Does
A ChatGPT-style chat window. User types their offer + ICP + goals → agent autonomously:
1. Researches ICP (job titles, company types, pain points, channels)
2. Builds lead lists using configured integrations (Apollo, Apify, Hunter, PhantomBuster, HubSpot, Google Sheets)
3. Generates personalized message sequences per channel via Super DM Setter AI
4. Creates campaigns in **draft** status
5. Sets up comment-to-DM automations in **draft** status
6. Presents structured **Outreach Plan** for human review + approval before launch

### Architecture
```
User intake form (offer, ICP, goals, channels)
    ↓
POST /api/super-agent/sessions  →  credit check (50 credits flat), insert session
    ↓
GET /api/super-agent/stream     →  SSE streaming agent loop
    ↓
Claude Opus 4.7 + adaptive thinking, tool-use loop:
  → research_icp          (AI sub-call, structured ICP analysis)
  → build_lead_list       (scrape existing integrations, persist list)
  → generate_sequences    (Super DM Setter AI per channel)
  → create_campaign       (draft status only)
  → create_comment_automation  (draft status only)
    ↓
plan_ready SSE event → OutreachPlan JSON
    ↓
User reviews, toggles items, clicks "Launch Selected"
    ↓
POST /api/super-agent/approve   →  activates campaigns + automations
```

### New DB Tables (migration 013)
**`super_agent_sessions`**
- id, user_id, offer, icp, goals, channels TEXT[], status, outreach_plan JSONB
- created_list_ids UUID[], created_campaign_ids UUID[], created_automation_ids UUID[]
- input_tokens, output_tokens (billing transparency)

**`super_agent_messages`**
- id, session_id, user_id, role, content JSONB (raw Anthropic ContentBlock array)
- display_type, display_data JSONB (for UI rendering)
- Append-only — used to replay conversation history to Claude + render chat log

### New API Routes
| Route | Purpose |
|---|---|
| `POST /api/super-agent/sessions` | Create session, deduct 50 credits |
| `GET /api/super-agent/stream?sessionId=` | SSE agent loop (maxDuration: 300) |
| `POST /api/super-agent/approve` | Launch approved campaigns + automations |
| `GET /api/super-agent/sessions/[id]` | Hydrate session for resume |

### SSE Events
| Event | Payload |

| `plan_ready` | `{ plan: OutreachPlan }` |
| `credits_update` | `{ remaining }` — after each tool deduction |
| `done` | `{ sessionId, status }` |
