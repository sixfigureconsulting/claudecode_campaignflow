## Feature 1 — Super DM Setter

### Overview
A new top-level page (`/super-dm-setter`) that acts as an AI co-pilot for appointment setters and DM closers. Trained on Speed to Lead, objection handling, closing frameworks, and channel-specific formatting rules.

### Files Created
- `src/app/(dashboard)/super-dm-setter/page.tsx` — Server page with hero header, stat pills, speed-to-lead urgency banner
- `src/components/super-dm-setter/SuperDMSetterClient.tsx` — Full interactive client component
- `src/app/api/super-dm-setter/generate/route.ts` — AI generation endpoint (Anthropic + OpenAI, rate-limited)
- `src/components/dashboard/Sidebar.tsx` — Added "Super DM Setter" nav item with "NEW" badge

### UI Layout (2-step vertical flow, max-width centered)

**Step 1 — Configure Your DM Agent** (collapsible once confirmed):
- Industry: Real Estate, High-Ticket Sales, DM Sales, Agency, Coaching, SaaS, Other
- Channel: LinkedIn DM, Instagram DM, SMS, WhatsApp, Twitter/X, Facebook DM, Cold Email
  - Each channel shows its optimal message length
- Message Type (8 options in colored grid):
  - Opener, Discover Pain, Poke the Bear, Pitch Offer, Handle Objection, Follow-Up, Urgency Close, Book Appointment
- Tone & Style: Professional, Casual, Direct & Bold, Empathetic, Urgency-Driven
- AI Model: Claude (Anthropic) or GPT-4o (OpenAI)
- **Auto-detects missing API keys** on page load — amber warning + disabled "Continue" button if key missing

**Step 2 — Add Your Conversation** (3 tabs):
1. **Paste Conversation** — text area, newest messages at bottom
2. **Drop Screenshot** — drag/drop or click, Claude Vision reads conversation from image
3. **CSV Leads** — upload CSV (max 50 leads), generates personalized openers in parallel batches of 3 with live progress

### AI System Prompt Covers
- Speed to Lead principle (5× more bookings in 5 mins)
- Pattern Interrupt for openers
- One CTA per message rule
- Channel-specific formatting (SMS = 1-2 sentences, Twitter = 1 sentence, LinkedIn = 3-5 sentences, etc.)
- 6 industry playbooks (Real Estate, High-Ticket, DM Sales, Agency, Coaching, SaaS)
- 8 message type frameworks (Feel-Felt-Found, Pain Bridge, Scarcity, etc.)
- Conversation temperature scoring (Cold 0-25 / Warm 26-55 / Hot 56-80 / Fire 81-100)
- Call trigger logic at 56+ score

### Results
- Heat Meter — animated bar with Cold → Warm → Hot → FIRE labels + score/100
- Reason for temperature score (2-sentence AI analysis)
- **Call-to-Action Banner** (appears at Hot/Fire) — pulsing red alert prompting: Call Now / Voice Note / Loom Video, with the exact reason why now is the right moment
- 3 ranked messages (Best / 2nd / 3rd) each showing:
  - Strategy name badge
  - Conversion probability (High/Medium/Low)
  - Copy-paste ready message
  - Why it works (1 sentence)
  - Character count vs channel limit
  - One-click copy button

### Visual Design
- Deep dark theme: `#0f0b25` indigo-black base
- Violet/indigo gradient accents on active states, buttons, glows
- Each message type card has its own color glow when selected (violet = Opener, amber = Poke the Bear, red = Urgency, green = Pitch, etc.)
- Three input tabs each glow in their own color (indigo = Paste, violet = Screenshot, emerald = CSV)

- Pulsing red "Live Intent Detection" indicator in header

- Passes `configuredProviders: string[]` to client
- Step 1 shows amber `!` badge on model buttons that have no key

## Feature 2 — Lists Lead Builder

### Overview
Integrated the full lead scraping system (previously only in campaigns Step 1) directly into the Lists page as a standalone builder. Users can now build lists without creating a campaign.

### Files Modified/Created
- `src/app/(dashboard)/lists/page.tsx` — Now fetches connected integrations server-side, passes to panel
- `src/components/lists/ListsPanel.tsx` — Full rewrite with embedded `ListBuilder` component
- `src/app/api/lists/scrape/route.ts` — New scraping endpoint (no `projectId` required, uses global keys)

### How It Works
Click **"+ New List"** → inline builder expands inside the page (no modal):

**Step 1 — Source picker**
- 6 integration cards: Apollo.io, Apify, PhantomBuster, Hunter.io, HubSpot CRM, Google Sheets
- Green "✓ Connected" badge if key is already in Settings → Integrations
- CSV Upload option below (separate section)

**Step 2a — API source input**
- URL / Domain / List ID input with per-source hint text
- Amber warning if key missing with link to Settings
- Enter key submits
- Calls `/api/lists/scrape` (new endpoint)

**Step 2b — CSV upload**
- Drag-and-drop or click zone
- Parsed client-side with same lenient COL_MAP as campaigns (handles Apollo, ZoomInfo, Lusha, Sales Nav formats)

**Step 3 — Preview + Save**
- Scrollable table: Name / Title / Company / Email (50 rows max preview)
- Pre-filled list name (source + date, or filename for CSV)
- Save → POST `/api/lists` → appears in grid immediately

### Scrape API (`/api/lists/scrape`)
- Auth + rate limit (20/min)
- No `projectId` needed — uses `getGlobalApiConfig()` (global keys from Settings → Integrations)
- Supports: Apollo (pagination up to 500), Apify (dataset URL), PhantomBuster (agent ID from URL + S3 fallback), Hunter (pagination up to 500), HubSpot (list memberships + batch contact fetch), Google Sheets (server-side CSV export)

## Credit System — Design Plan (NOT YET BUILT)

### Principle
Credits are charged for **platform-orchestrated actions**. Bringing your own API key does not make an action free — you pay the API cost separately, credits cover platform orchestration.

### What Is FREE (0 credits)
- CSV upload / download / storage
- Google Sheets import
- All read operations (lists, reports, dashboard)
- Creating clients, projects, reports
- Lead fetch from Apollo, Apify, PhantomBuster, Hunter, HubSpot (user pays their own API)
- Saving / managing lists

### Credit Costs Per Action
| Action | Credits | Notes |
|---|---|---|
| Campaign execution — per lead pushed | 1/lead | Core action, highest volume driver |
| Lead qualification (AI ICP filter) — per lead | 2/lead | AI call per lead (user's OpenAI pays API) |
| Sequence generation — per lead | 3/lead | Multiple AI calls per lead |
| Apollo enrichment — per lead | 1/lead | Company website lookup per record |
| Exclusion check — per lead | 0.5/lead | Instantly/HubSpot duplicate check |
| Super DM Setter — single conversation | 5/generation | 3 messages + intent scoring |
| Super DM Setter — screenshot analysis | 8/generation | Vision AI on top |
| Super DM Setter — CSV batch | 3/lead | Personalized opener per prospect |
| AI report recommendation | 10/report | Full strategic analysis |
| SFC sequence + push | 15/run | Anthropic + Heyreach/Instantly in one shot |

### Plan Tiers
| Plan | Price | Credits/month |
|---|---|---|
| Starter | $97/m | 10,000 |
| Growth | $197/m | 25,000 |
| Agency | $397/m | 75,000 |
| Top-up | — | $19 / 5,000 credits |

### 10k Credits Sanity Check (Starter plan)
- Active agency: 3k leads pushed + 1k qualified + 500 sequences + 50 DM Setter = 6,780 credits ✓
- DM-heavy user: 500 leads + 200 DM Setter + 50 CSV batch = 1,680 credits ✓
- Power user: 8k leads pushed + 200 qualified = 8,400 credits (natural upgrade trigger) ✓

### Build Plan (when ready)
1. **DB migration** — 2 new tables:
   - `user_credits` (user_id, balance, plan_credits, period_start, period_end)
   - `credit_transactions` (user_id, action, credits_used, metadata, created_at)
2. **Stripe webhook** — reset credits on `invoice.payment_succeeded` / new subscription
3. **Utility** — `deductCredits(userId, action, amount, metadata)` — atomic check+deduct
4. **Wire into ~8 routes** — check balance before, deduct after successful response only:
   - `push-leads`, `qualify-leads`, `generate-sequences`, `apollo-enrich`, `check-exclusions`, `sfc-sequence`, `ai/recommend`, `super-dm-setter/generate`
5. **TopBar credits badge** — live balance display
6. **Low-credit warning** — toast at <500 credits, banner at <100
7. **Credits page / breakdown** — transaction history table
8. **Top-up flow** — separate Stripe checkout type for credit packs

### Key Technical Note
Deduction must happen AFTER successful API response — never deduct on failure. Use try/finally pattern: deduct in `finally` only if `success === true`. This prevents credit loss on downstream API errors.
