## Feature 2 ‚Äî Social DM Outreach Infrastructure

### DB Migrations (written 2026-04-08, confirmed present)
- `supabase/migrations/009_social_dm.sql` ‚Äî `social_campaigns`, `social_campaign_leads`, `oauth_connections` + RLS + triggers

> **Action required:** Run `supabase link` then `supabase db push` to apply migrations 009 + 010 to the live database.

### New API routes

**`src/app/api/social/campaigns/route.ts`** ‚Äî `GET` / `POST`
- GET: returns all social campaigns for the authenticated user
- POST: creates a campaign record, verifies list ownership, copies up to 500 contacts from `lead_list_contacts` into `social_campaign_leads`

**`src/app/api/social/campaigns/[id]/launch/route.ts`** ‚Äî `POST`
- Fetches pending leads (capped at 100 per launch call)
- Pre-flight credit check against `user_credits.balance`
- **LinkedIn** ‚Üí pushes leads to Heyreach via `AddLeadsToList` (creates list named after campaign if needed)
- **Reddit** ‚Üí loops leads, calls `/api/social/send/reddit` per lead
- **Other channels** (Twitter, Email, Instagram, Facebook) ‚Üí marks campaign `active`, cron will handle actual send
- Deducts `social_dm_send` credits (1 per sent DM) via `deductCredits()`
- Updates `social_campaigns.sent_count` + `failed_count`

**`src/app/api/social/send/reddit/route.ts`** ‚Äî `POST`
- Loads the user's Reddit OAuth connection from `oauth_connections`
- Auto-refreshes expired access token using `refresh_token` (stores refreshed token back to DB)
- Calls Reddit `POST /api/compose` to send private message
- Updates `social_campaign_leads` status ‚Üí `sent` / `failed` with timestamp and platform message ID
- Increments `social_campaigns.sent_count` directly (no RPC needed)

### Reddit OAuth flow

**`src/app/api/oauth/reddit/route.ts`** ‚Äî `GET`
- Initiates OAuth 2.0 PKCE-style flow
- Scopes: `privatemessages identity`
- Duration: `permanent` (gets refresh token)
- CSRF state: HMAC-signed `user_id:timestamp` pair

**`src/app/api/oauth/reddit/callback/route.ts`** ‚Äî `GET`
- Exchanges auth code for tokens
- Fetches Reddit username via `/api/v1/me`
- Encrypts both `access_token` and `refresh_token` with `encryptApiKey()` (AES-256, same as API keys)
- Upserts into `oauth_connections` on `(user_id, platform)` conflict
- Redirects to `/settings?reddit_oauth=success|denied|error`

**`src/app/api/oauth/connections/route.ts`** ‚Äî `GET` / `DELETE`
- GET: lists all OAuth connections for the user (platform, username, scopes, expiry ‚Äî no tokens)
- DELETE `?platform=reddit`: removes the connection row

### Feature 8 ‚Äî Twitter/X OAuth + DM send

**Created:** `src/app/api/oauth/twitter/route.ts`
- PKCE flow: generates `code_verifier` + `code_challenge` (SHA256/base64url)
- Stores `code_verifier` in `tw_cv` httpOnly cookie (10 min TTL)
- CSRF state: same HMAC-signed `user_id:timestamp` pattern as Reddit
- Scopes: `tweet.read users.read dm.read dm.write offline.access`

**Created:** `src/app/api/oauth/twitter/callback/route.ts`
- Reads `code_verifier` from `tw_cv` cookie
- Exchanges code for tokens via Twitter OAuth 2.0
- Fetches `@username` + user ID from `/2/users/me`
- Encrypts both tokens; upserts into `oauth_connections` on `(user_id, platform)`
- Redirects to `/settings?twitter_oauth=success|denied|error`
- Clears `tw_cv` cookie on success

**Created:** `src/app/api/social/send/twitter/route.ts`
- Auto-refreshes expired access token (60s buffer), re-stores to DB
- Resolves username ‚Üí user ID if only handle provided (via `/2/users/by/username/`)
- Sends DM via `POST /2/dm_conversations/with/:participant_id/messages`
- Updates `social_campaign_leads.status` + increments `social_campaigns.sent_count`

**Modified:** `src/app/api/social/campaigns/[id]/launch/route.ts`
- Added `channel === "twitter"` branch ‚Äî loops leads, calls `/api/social/send/twitter` per lead
- Skips leads with no `twitter_handle`
