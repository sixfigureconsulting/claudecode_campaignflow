### 2. Instantly Campaign Picker on Sync
**Goal:** When syncing from Instantly, let the user pick a specific campaign instead of aggregating all campaigns into one blob of metrics.

**Flow:**
1. User clicks "Sync from Instantly" → button shows "Loading campaigns…"
2. Frontend fetches `GET /api/reports/sync/campaigns?tool=instantly`
3. Picker UI appears — native `<select>` listing all campaigns (paused/stopped labeled)
4. User picks one → clicks "Sync Selected Campaign"
5. Sync calls `POST /api/reports/sync` with `{ reportId, tool, campaignId, campaignName }`
6. Backend calls `GET /api/v2/campaigns/analytics?id={campaignId}` (Instantly API)
7. Success message: "Synced 6 metrics from Instantly — My Campaign Name"

Other tools (Smartlead, HeyReach, Lemlist) still aggregate all campaigns — per-campaign picker can be added later via `PICKER_TOOLS` set.

**Files:**
- `src/app/api/reports/sync/campaigns/route.ts` _(new)_ — fetches campaign list for any connected tool
- `src/app/api/reports/sync/route.ts` — updated `syncInstantly()` to accept optional `campaignId`/`campaignName`; route handler passes them through
- `src/components/reports/SyncReportCard.tsx` — full rewrite with 3-phase state machine: `idle → loadingCampaigns → picking → syncing → success/error`
